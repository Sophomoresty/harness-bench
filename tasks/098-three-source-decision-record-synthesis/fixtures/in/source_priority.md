# Source Priority

Use this priority order when sources conflict:

1. Formal security, legal, and finance documents govern their scoped domains.
2. Later decision emails govern schedule and coordination when they explicitly say a decision changed.
3. Product requirements define proposed product scope unless blocked by security, legal, or finance.
4. Meeting transcript is useful for context and action owners but does not by itself create final approval.

If a source lacks authority for a domain, preserve it as background or a rejected proposal.
